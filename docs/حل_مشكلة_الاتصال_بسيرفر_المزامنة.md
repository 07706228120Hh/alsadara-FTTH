### الخطوة 1: إنشاء Entity — CompanyFtthCredential.cs
```
المسار: src/Backend/Core/Sadara.Domain/Entities/CompanyFtthCredential.cs
```
الحقول المتوقعة:
- Id (int, PK)
- CompanyId (Guid, FK)
- FtthUsername (string)

- FtthServerUrl (string, default: admin.ftth.iq)
- AutoSyncEnabled (bool)
- SyncIntervalHours (int)
- SyncStartHour (int, default: 6)
- SyncEndHour (int, default: 23)
- CreatedAt, UpdatedAt

الحقول المتوقعة:

### الخطوة 5: إنشاء Migration
```bash
dotnet ef migrations add AddCompanyFtthSettingsAndSyncLogs
```

### الخطوة 6: إنشاء CompanyFtthSettingsController.cs
```
المسار: src/Backend/API/Sadara.API/Controllers/CompanyFtthSettingsController.cs
Route: api/company-ftth-settings
Auth: [ApiKeyOrJwtAuth]
```

الـ endpoints المطلوبة:
1. `GET /{companyId}` — جلب إعدادات الشركة
2. `POST /` — حفظ/تحديث الإعدادات
3. `POST /{companyId}/test` — اختبار الاتصال بسيرفر FTTH
4. `GET /{companyId}/sync-status` — حالة المزامنة (عدد المشتركين، آخر مزامنة)
5. `POST /{companyId}/trigger-sync` — تشغيل مزامنة يدوية
6. `POST /{companyId}/cancel-sync` — إلغاء مزامنة جارية
7. `GET /{companyId}/sync-logs` — سجل المزامنات
8. `DELETE /{companyId}/sync-logs/{id}` — حذف سجل واحد
9. `DELETE /{companyId}/sync-logs` — حذف كل السجلات
10. `GET /{companyId}/missing-stats` — البيانات الناقصة
11. `POST /{companyId}/refetch-missing` — إعادة جلب البيانات الناقصة

### الخطوة 7: نشر على السيرفر

```bash
# بناء

scp -r bin/Release/net9.0/linux-x64/publish/* root@72.61.183.61:/path/to/api/

# تطبيق Migration
dotnet ef database update
```
