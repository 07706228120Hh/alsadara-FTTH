# تقرير تحليل منصة الصدارة (Sadara Platform)

**تاريخ التقرير**: 6 فبراير 2026

---

## الفكرة العامة

منصة **الصدارة** هي نظام متكامل لإدارة شركات خدمات الإنترنت الفايبر (FTTH - Fiber To The Home) في العراق. تخدم **ثلاثة أطراف** من خلال **ثلاث واجهات** مختلفة، وكلها تتصل بخادم API مركزي واحد.

---

## المكونات الثلاثة بالتفصيل

### 1. خادم API الخلفي (.NET 9)

📍 المسار: `src/Backend/`

#### البنية المعمارية (Clean Architecture)

الخادم مقسم إلى **4 طبقات** بتبعيات واضحة:

| الطبقة | المسار | المسؤولية |
|--------|--------|-----------|
| **Domain** | `Core/Sadara.Domain/` | الكيانات، التعدادات، الواجهات — لا تعتمد على أي شيء |

كل كيان يرث من `BaseEntity<TId>` الذي يوفر: `CreatedAt`, `UpdatedAt`, `IsDeleted`, `DeletedAt`

| الكيان | الوصف | المعرّف |
|--------|-------|---------|

| **Company** | شركة إنترنت — لها اشتراك، ميزات مفعلة، حد مستخدمين، ربط بنظام المواطن | `Guid` |

| **CitizenSubscription** | اشتراك مواطن في باقة — بحالات متعددة وعنوان تركيب | `Guid` |
| **ServiceRequest** | طلب خدمة (تركيب، صيانة، نقل) — مع تعليقات ومرفقات وتاريخ حالات | `Guid` |
| **SupportTicket** | تذكرة دعم فني من المواطن — بأولوية وتصنيف وتقييم | `Guid` |

| **Service / ServiceOperation** | تعريف الخدمات وعملياتها | `int` |
| **Notification / Advertising / AppVersion** | إشعارات، إعلانات، إصدارات التطبيق | `long` / `int` |

#### نظام الأدوار (UserRole) — 9 أدوار هرمية

```
SuperAdmin (99)        ← مدير النظام الكامل
  └─ CompanyAdmin (20) ← مدير شركة
       └─ Manager (14) ← مشرف
            └─ TechnicalLeader (13) ← ليدر فني
                 └─ Technician (12) ← فني
                 └─ Viewer (11)     ← مشاهد فقط
                 └─ Employee (10)   ← موظف عادي
Merchant (1)           ← تاجر (نظام التجارة الإلكترونية)
Citizen (0)            ← مواطن/مشترك
```

- **V1 (قديم)**: أعلام `true/false` بسيطة — `{"attendance":true,"agent":false}`
- **V2 (الحالي)**: مبني على الإجراءات — `{"users":{"view":true,"add":true,"edit":false,"delete":false}}`
- **قوالب صلاحيات** (`PermissionTemplate`): مجموعات صلاحيات جاهزة يمكن تطبيقها على الموظفين

- **قاعدة البيانات**: PostgreSQL (مع fallback لـ InMemory في التطوير)

- **Seed Data**: بيانات أولية تلقائية (Super Admin، بيانات تجريبية في التطوير)

## تدفق العمليات الأساسية

### تسجيل دخول موظف شركة
```
Flutter Desktop → POST /api/Auth/login (CompanyCode + Username + Password)
              → JWT Token ← الخادم
              → تحميل صلاحيات المستخدم (V2 JSON)
              → عرض الصفحات حسب الصلاحيات (PermissionsGate)
```

### تسجيل مواطن جديد
```
Citizen Web → POST /api/CitizenAuth/register (Phone + Password + Name)
          → إرسال رمز تحقق SMS
          → POST /api/CitizenAuth/verify-phone
          → JWT Token ← الخادم
          → ربط المواطن بالشركة المرتبطة بنظام المواطن
```

### طلب اشتراك إنترنت
```
Citizen Web → GET /api/InternetPlans (عرض الباقات)
          → POST /api/CitizenSubscriptions (طلب اشتراك)
          → الشركة تتلقى إشعار في Flutter Desktop
          → الموظف يراجع ويعيّن فني
          → الفني ينفذ التركيب
          → تحديث الحالة → المواطن يتلقى إشعار
```

### نظام طلبات الخدمة
```
ServiceRequest: Pending → Reviewing → Approved → Assigned → InProgress → Completed
                                    → Rejected
                         → OnHold
                         → Cancelled
```

## نظام Multi-Tenant

- كل **شركة** (Company) هي tenant منفصل
- الموظفون مرتبطون بشركة عبر `User.CompanyId`
- المواطنون مرتبطون بشركة عبر `Citizen.CompanyId`
- شركة واحدة فقط يمكن ربطها بنظام المواطن (`IsLinkedToCitizenPortal = true`)
- كل شركة لها خدمات مفعلة (`CompanyService`)، صلاحيات، وحد مستخدمين

## 🔴 مشاكل حرجة (P0) — يجب إصلاحها فوراً

### 1. مفتاح JWT الافتراضي ثابت في الكود
**الملفات**: `src/Backend/API/Sadara.API/Program.cs` (السطر 51)

```csharp
// إذا لم يوجد إعداد JWT:Secret، يُستخدم هذا المفتاح الثابت:
"YourSuperSecretKeyThatIsAtLeast32CharactersLong!"
```
**الخطر**: إذا حدث خطأ في إعدادات الإنتاج، يمكن لأي شخص تزوير JWT tokens والدخول كـ SuperAdmin.

**الحل**: رفض تشغيل التطبيق إذا لم يُعيّن `Jwt:Secret`:
```csharp
throw new InvalidOperationException("JWT Secret not configured");
```

**الملف**: `src/Backend/API/Sadara.API/Controllers/CitizenAuthController.cs`

debugOtp = otp // للتطوير فقط — لكنه يعمل في الإنتاج أيضاً!
