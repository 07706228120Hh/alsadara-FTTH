# نظام المحاسبة - Sadara Accounting System

## نظرة عامة

نظام محاسبي متكامل مبني داخل منصة الصدارة لإدارة العمليات المالية لشركات الإنترنت (ISP).
يعتمد على مبدأ **القيد المزدوج** (Double Entry) حيث كل عملية مالية تُسجَّل كقيد محاسبي متوازن (مدين = دائن).

---

## المكونات الرئيسية

### 1. شجرة الحسابات (Chart of Accounts) — `Account`

الحساب المحاسبي هو الوحدة الأساسية في النظام. يدعم **التسلسل الهرمي** (حساب أب ← حسابات فرعية).

#### أنواع الحسابات:

| النوع | الوصف | أمثلة | الطبيعة |
|-------|-------|-------|---------|
| **أصول** (Asset) | ما تملكه الشركة | نقدية، بنك، معدات، ذمم مدينة | مدين |
| **خصوم** (Liability) | ما على الشركة من التزامات | قروض، ذمم دائنة، رواتب مستحقة | دائن |
| **حقوق ملكية** (Equity) | رأس مال الشركة | رأس المال، أرباح محتجزة | دائن |
| **إيرادات** (Revenue) | دخل الشركة | اشتراكات إنترنت، رسوم تركيب | دائن |
| **مصروفات** (Expense) | ما تنفقه الشركة | رواتب، إيجار، وقود، صيانة | مدين |

#### هيكل الترقيم:
```
1000  ← أصول (مستوى 1 - رئيسي)
├── 1100  ← نقدية وما يعادلها (مستوى 2)
│   ├── 1110  ← الصندوق الرئيسي (مستوى 3 - فرعي نهائي)
│   ├── 1120  ← البنك (مستوى 3)
│   └── 1130  ← القاصة (مستوى 3)
├── 1200  ← ذمم مدينة
│   └── 1210  ← تحصيلات فنيين غير مسلمة
2000  ← خصوم
├── 2100  ← رواتب مستحقة
3000  ← حقوق ملكية
4000  ← إيرادات
├── 4100  ← إيرادات اشتراكات
├── 4200  ← إيرادات تركيب
5000  ← مصروفات
├── 5100  ← رواتب وأجور
├── 5200  ← إيجار
├── 5300  ← وقود ومواصلات
```

#### خصائص مهمة:
- **`IsLeaf`**: فقط الحسابات النهائية (الأوراق) يمكن الترحيل إليها مباشرة
- **`IsSystemAccount`**: حسابات نظامية لا يمكن حذفها
- **`CurrentBalance`**: يتحدث تلقائياً مع كل قيد يُرحَّل
- **`Level`**: مستوى العمق في الشجرة (1 = جذر)

### 2. القيد المحاسبي (Journal Entry) — `JournalEntry` + `JournalEntryLine`

القيد هو **قلب النظام المحاسبي**. كل عملية مالية تُترجَم إلى قيد.

#### القاعدة الذهبية:
$$\sum \text{مدين} = \sum \text{دائن}$$

#### دورة حياة القيد:
```
مسودة (Draft) ──→ مرحَّل (Posted) ──→ ملغي (Voided)
                        ↓
                  تحديث أرصدة الحسابات
```

- **مسودة**: قيد محفوظ لم يؤثر على الأرصدة بعد
- **مرحَّل**: تم اعتماده وتحديث الأرصدة — لا يمكن تعديله
- **ملغي**: تم عكس تأثيره على الأرصدة

#### أنواع المراجع (`ReferenceType`):
| النوع | الوصف |
|-------|-------|
| `Manual` | قيد يدوي أنشأه المحاسب |
| `CashTransaction` | ناتج عن حركة صندوق |
| `Salary` | ناتج عن صرف رواتب |
| `TechnicianCollection` | ناتج عن تسليم تحصيل فني |
| `Expense` | ناتج عن تسجيل مصروف |

#### مثال - قيد صرف راتب:
| الحساب | مدين | دائن |
|--------|------|------|
| 5100 - رواتب وأجور | 500,000 | |
| 1110 - الصندوق الرئيسي | | 500,000 |

---

### 3. الصندوق / القاصة (Cash Box) — `CashBox` + `CashTransaction`

يتتبع **النقد الفعلي** في الشركة.

#### أنواع الصناديق:
| النوع | الوصف |
|-------|-------|
| **رئيسي** (Main) | الصندوق الرئيسي للشركة |
| **قاصة** (PettyCash) | صندوق مصاريف نثرية |
| **بنك** (Bank) | حساب بنكي |

#### العمليات:
- **إيداع** (`Deposit`): إدخال مبلغ للصندوق ← يزيد الرصيد
- **سحب** (`Withdrawal`): إخراج مبلغ من الصندوق ← ينقص الرصيد

#### كل حركة تُسجَّل:
- مبلغ الحركة
- الرصيد بعد الحركة (`BalanceAfter`)
- المستخدم الذي أجراها
- القيد المحاسبي التلقائي

#### ربط الصندوق بشجرة الحسابات:
كل صندوق يُربط بحساب في شجرة الحسابات (`LinkedAccountId`)، مما يضمن أن رصيد الصندوق = رصيد الحساب المحاسبي.

---

### 4. الرواتب (Salaries) — `EmployeeSalary`

سجل **شهري** لكل موظف في الشركة.

#### حساب صافي الراتب:
$$\text{صافي الراتب} = \text{أساسي} + \text{بدلات} + \text{مكافآت} - \text{خصومات}$$

#### دورة الحياة:
```
توليد الرواتب للشهر ──→ مراجعة وتعديل ──→ صرف (دفع)
      (Pending)              (Pending)         (Paid)
```

#### العمليات:
1. **توليد الرواتب**: يأخذ الراتب الأساسي من بيانات الموظف ويُنشئ سجل لكل موظف
2. **تعديل**: إضافة بدلات، خصومات، مكافآت قبل الصرف
3. **صرف فردي**: دفع راتب موظف واحد ← قيد محاسبي تلقائي
4. **صرف جماعي**: دفع جميع الرواتب المعلقة دفعة واحدة

#### القيد التلقائي عند الصرف:
| الحساب | مدين | دائن |
|--------|------|------|
| رواتب وأجور (مصروف) | المبلغ | |
| الصندوق الرئيسي (أصل) | | المبلغ |

---

### 5. تحصيلات الفنيين (Technician Collections) — `TechnicianCollection`

المبالغ التي **يجمعها الفنيون ميدانياً** من المواطنين (اشتراكات، رسوم تركيب...).

#### دورة الحياة:
```
تسجيل التحصيل ──→ مراجعة ──→ تسليم للصندوق
  (غير مسلم)                    (مسلم)
```

#### البيانات المسجلة:
- الفني الذي حصّل
- المواطن (اختياري)
- طلب الخدمة (اختياري)
- المبلغ + طريقة الدفع (نقدي/بطاقة/تحويل)
- رقم الإيصال

#### عند التسليم:
1. يُحدَّث `IsDelivered = true` و `DeliveredAt`
2. يُحدَّد الصندوق المستلم (`CashBoxId`)
3. يُنشأ حركة إيداع في الصندوق
4. يُنشأ قيد محاسبي تلقائي:

| الحساب | مدين | دائن |
|--------|------|------|
| الصندوق (أصل) | المبلغ | |
| إيرادات اشتراكات | | المبلغ |

---

### 6. المصروفات (Expenses) — `Expense`

مصروفات الشركة التشغيلية والعامة.

#### الفئات المتاحة:
| الفئة | الوصف |
|-------|-------|
| عام (General) | مصروفات متنوعة |
| إيجار (Rent) | إيجار مقر الشركة |
| خدمات (Utilities) | كهرباء، ماء |
| رواتب (Salaries) | رواتب إضافية |
| صيانة (Maintenance) | صيانة معدات وأجهزة |
| معدات (Equipment) | شراء معدات جديدة |
| وقود (Fuel) | وقود ومواصلات |
| إنترنت (Internet) | اشتراكات إنترنت |
| تسويق (Marketing) | حملات تسويقية |

#### عند التسجيل:
- يُحدَّد الحساب المحاسبي (من شجرة الحسابات - نوع مصروف)
- يُحدَّد الصندوق الذي دُفع منه (اختياري)
- يُرفق إيصال (اختياري)
- يُنشأ قيد محاسبي تلقائي

## التقارير المتاحة

### 1. لوحة القيادة (Dashboard)
- إجمالي الأصول والخصوم
- إجمالي الإيرادات والمصروفات
- رصيد الصناديق
- التحصيلات غير المسلمة

### 2. كشف حساب (Account Statement)
- حركات حساب معين خلال فترة محددة
- رصيد أول المدة وآخر المدة

### 3. ميزان المراجعة (Trial Balance)
- أرصدة جميع الحسابات
- التحقق: مجموع المدين = مجموع الدائن

### 4. ملخص تحصيلات الفنيين
- إجمالي تحصيلات كل فني
- المسلم وغير المسلم

## قاعدة البيانات

### الجداول (PostgreSQL):
```
Accounts              ← شجرة الحسابات
JournalEntries        ← القيود المحاسبية
JournalEntryLines     ← سطور القيود
CashBoxes             ← الصناديق
CashTransactions      ← حركات الصناديق
EmployeeSalaries      ← الرواتب
TechnicianCollections ← تحصيلات الفنيين
Expenses              ← المصروفات
```

### دقة الأرقام المالية:
جميع الحقول المالية تستخدم `decimal(18,2)` — أي 18 رقم إجمالي و 2 رقم عشري.

### الحذف الناعم:
جميع الكيانات ترث `BaseEntity` الذي يحتوي `IsDeleted`. عند الحذف يُعلَّم السجل كمحذوف ولا يُحذف فعلياً. الاستعلامات تستبعد المحذوفات تلقائياً عبر Query Filters.

### الفهارس:
39 فهرس تم إنشاؤها لتحسين الأداء على الحقول الأكثر استخداماً (CompanyId, Status, EntryDate, إلخ).
